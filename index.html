Alrededor de los EE.UU. – Detalles del proyecto

Contexto del proyecto

Este proyecto forma parte del Sprint 11 del bootcamp de Desarrollo Web de Tripleten. El objetivo fue refactorizar el sitio “Alrededor de los EE.UU.” para utilizar programación orientada a objetos (POO) en JavaScript y consumir una API REST para gestionar la información del usuario y las tarjetas.

El sitio permite editar el perfil, añadir nuevas tarjetas con imágenes y títulos, dar “me gusta”, eliminar tarjetas y visualizar imágenes en tamaño completo mediante ventanas emergentes (popups).

Mi análisis

El principal reto fue pasar de un código procedural a una arquitectura basada en clases ES6 (Card, Section, Popup, PopupWithImage, PopupWithForm, PopupWithConfirmation, UserInfo, FormValidator y Api). Esto implicó pensar en términos de responsabilidades y acoplamiento débil, separando la lógica de UI, la validación de formularios y las llamadas a la API en módulos independientes.

También afiné la validación de formularios usando las capacidades nativas de HTML5 y clases de error, además de gestionar estados de carga en los botones al enviar formularios. Trabajé con promesas y fetch para sincronizar la interfaz con el servidor, incluyendo la carga inicial de usuario y tarjetas, los likes y la actualización del avatar.

Conclusiones principales

  • Comprendí mejor cómo estructurar aplicaciones front-end usando clases, módulos ES6 y principios de POO.
  • Fortalecí mi manejo de Fetch API y el trabajo con endpoints REST para operaciones CRUD.
  • Mejoré la experiencia de usuario con validación activa, popups accesibles y estados de carga en los formularios.
  • Este proyecto demuestra mi capacidad para mantener código organizado, escalable y alineado con buenas prácticas modernas.

De patria a patria – Detalles del proyecto

Contexto del proyecto

Este proyecto corresponde al Sprint 6 del bootcamp de Desarrollo Web. El objetivo fue ampliar el sitio del Sprint 5 añadiendo nuevas secciones (photo-grid y places) y lograr un diseño 100% responsivo para las resoluciones de 320px, 768px y 1280px, partiendo de un diseño en Figma.

El sitio presenta una galería fotográfica y descripciones de ciudades relacionadas con los autores, con imágenes optimizadas y un layout que se adapta suavemente a resoluciones intermedias.

Mi análisis

El principal foco técnico fue dominar CSS Grid y Flexbox para construir una maquetación flexible que respetara las especificaciones de Figma en distintas resoluciones. Implementé una estructura de archivos basada en la metodología BEM, con CSS separado por bloques, lo que mejora la mantenibilidad y la escalabilidad del proyecto.

Presté atención a la tipografía (fuentes locales Inter y Noto Serif), al uso de degradados, sombras y al recorte de imágenes con object-fit para mantener una apariencia consistente en todas las tarjetas y secciones.

Conclusiones principales

  • Consolidé mi experiencia en diseño responsivo usando Grid, Flexbox y media queries.
  • Aprendí a traducir un diseño de Figma a código pixel-perfect cuidando espaciados, tipografía e imágenes.
  • Refiné el uso de BEM y la organización de estilos en bloques, lo que facilita la lectura y el mantenimiento del CSS.
  • Este proyecto muestra mi capacidad para crear interfaces estáticas limpias, modernas y adaptables a múltiples dispositivos.
