Alrededor de los EE.UU. – Detalles

Contexto del proyecto

Proyecto del Sprint 11 del bootcamp Tripleten. El objetivo fue refactorizar el sitio para utilizar programación orientada a objetos (POO) en JavaScript y consumir una API REST para gestionar usuario y tarjetas.

Mi análisis

El reto principal fue pasar de un enfoque procedural a una arquitectura basada en clases (Card, Section, Popup, UserInfo, FormValidator, Api, etc.). Esto supuso separar responsabilidades, reducir acoplamiento y mantener la lógica de UI, validación y llamadas a la API en módulos independientes.

Conclusiones principales

  • Consolidé el uso de clases y módulos ES6 para organizar aplicaciones front-end.
  • Fortalecí el manejo de Fetch API y endpoints REST para operaciones CRUD.
  • Mejoré la UX con validación activa, popups reutilizables y estados de carga en formularios.

De patria a patria – Detalles

Contexto del proyecto

Proyecto del Sprint 6 del bootcamp Tripleten. El objetivo fue ampliar el sitio del Sprint 5 añadiendo secciones photo-grid y places y lograr un diseño 100% responsivo basado en Figma para 320px, 768px y 1280px.

Mi análisis

El foco técnico estuvo en aplicar y consolidar el uso de CSS Grid y Flexbox para construir una maquetación flexible que respetara las especificaciones de Figma. La estructura de estilos sigue la metodología BEM, con CSS separado por bloques en el proyecto original.

Conclusiones principales

  • Refuerzo de habilidades en diseño responsivo usando Grid, Flexbox y media queries.
  • Experiencia en llevar un diseño de Figma a código cuidando tipografía, espaciados e imágenes.
  • Organización de CSS con BEM para facilitar mantenimiento y escalabilidad.
